# Lifecycle Rules

## Capture To Memory

1. 新信息先进入 `inbox/`
2. 判断是否值得长期保存
3. 若值得长期保存，整理到 `memory/`
4. 若与当前目标相关，增加 `okr/` 关联

## Review To Action

1. review 记录当前进展和阻塞
2. 若出现稳定事实，回写 `memory/`
3. 若出现目标变化，更新 `okr/`

## Archive Rules

- 已失效但仍有参考价值的内容转为 `archived`
- 季度 OKR 结束后归档到 `okr/archive/` 或保留原季度目录并补充复盘
