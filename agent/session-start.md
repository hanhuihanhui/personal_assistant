# Session Start

每次进入这个仓库时，先用下面的顺序快速建立上下文。

## Step 1

读取：

- `memory/index.md`
- `reviews/index.md`
- `okr/index.md`

## Step 2

根据用户请求判断任务类型：

- 新信息采集: `inbox/` 与 `templates/inbox-capture.md`
- 长期信息更新: `memory/`
- 进度回顾: `reviews/`
- 目标规划与跟踪: `okr/`
- 方法或参考资料: `knowledge/`

## Step 3

执行时遵守：

- 新信息若暂不确定归属，先放 `inbox/`
- 稳定事实进入 `memory/`
- review 的结论应考虑同步到 `memory/` 或 `okr/`
- 目标变化应更新 `okr/`，必要时加 `related` 引用

## Step 4

输出时尽量告诉用户：

- 读了哪些文件
- 更新了哪些文件
- 哪些事项仍需要后续 review 或 OKR 跟进
